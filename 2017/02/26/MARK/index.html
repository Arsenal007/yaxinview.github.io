博客终建成

发表于 2017-02-26 | | 人阅读

纪念

经历了两天晚上的熬夜苦战（25号凌晨三点+ 26号凌晨两点）和各路大神的不吝赐教，终于在今天把想了好久的博客搭建完成。今天白天在图书馆优化了一些细节。终算初具规模。这么具有重要意义的时刻，确实该记上一笔。

为什么要搭建博客

其实从上大学直到现在，都有写日记的习惯。但日记记录的必然都是很私人的东西。自从开始接触前端，自己写代码以来（虽然也只有不到3个月），看了很多技术大牛的博客，获益匪浅。于是萌生了为何不搭一个自己的博客的想法。一来分享一个小白是如何从0到1，再从1到正无穷的心路历程。二来，等到自己对技术有所见解之后，也可以通过自己的平台分享给需要的人。还有就是想把自己经历的难忘的瞬间，用这种方式记录下来。有了自己亲手搭建的博客，应该会鞭策自己多进行高质量的文字输出吧。深知什么事情都贵在坚持，在这个知识碎片化的时代，平时接触到的东西，很需要去总结和思考，如果只是一个接受知识的人，很难形成自己独立思考的习惯，很多能力虽然看天赋，但也是需要自己努力去培养的，只希望自己坚持，而不是三分钟热度（虽然在不少事情上我确实是三分钟热度……），但是喜欢就可以很好的坚持。乐此不疲这个词，形容此次搭建过程再合适不过了。

GitHub+Hexo搭建博客过程中踩的坑

不知道cd是什么命令

我是真小白，不知道像我这样连Mac 终端里面cd命令都不知道是啥的人还多不多看。。。于是看到教程上知道，cd到自己的blog文件这句话就先不懂了。为了避免有人和我犯一样的错误，我在这里解释一下cd命令。

cd命令 是Linux的常用命令之一。命令功能是切换到当前目录。如cd blog即表示切换到名为blog的文件夹下。

因为很多操作不是在根目录下操作的，需要在自己建的博客文件下下操作，所以cd到自己的博客文件夹这一步，因为当时一直在根目录下操作，导致hexo s命令一直不成功。

于是，简单的四个命令可以预览博客效果啦~

$ cd blog

$ hexo init blog

$ npm install

$ hexo s

本地localhost:4000可以打开，用yaxinview.github.io却该页无法显示

我因为没有将自己这里github.io前面的 yaxinview一定要和你GitHub的用户名相同，而且尽量不要用大写。因为我一开始GitHub的用户名首字母是大写，而且没有和yaxinview一致，于是更换了GitHub的用户名为yaxinview，成功打开链接 yaxinview.github.io。 嗨森。

域名解析一直不成功

作为一个有自己域名的人，怎么能忍受只能通过yaxinview.hithub.io访问，不能通过我自己申请的域名访问呢？域名如何解析的步骤在网上现有的教程上也很多了，但是我一步步走就是总显示不成功。经过大神指点和我的多次尝试，才知道。教程上让把本地的CNAME记录放到 themes/../source下面。但是实践发现，要放到yaxinview/source下才能正常访问。难道是前面放的太深了？哈哈哈。最后可以通过自己域名访问的那一刻，简直要激动的跳起来啦~但是念在大晚上邻居们的感受，我还是低调点儿……
