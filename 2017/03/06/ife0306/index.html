重视基础，勤敲多练

发表于 2017-03-06 | | 人阅读

IFE HTML 基础练习

其实，发现很多js写的好的人，都不屑于或者不喜欢写HTML或者调CSS样式，估计是觉得太简单吧，今天心血来潮，决定先停下刷IFE的JS题，做几道基础的HTML，看看自己还是不是足够熟练，事实证明，有些属性还是忘了，但好在有W3C school，只要知道需要什么样式时，去查一查，基本就可达到想要的效果。

今天把零基础的HTML 任务一 和 任务二写完了。自己独立撸出这些行代码的感觉真好。对表格的合并单元格，input属性各个类型的使用，盒子模型的布局排版，阴影等都练到和巩固了。虽然现在框架多了，有些东西都是现成的，但是多敲敲代码，保持这种熟练，应该也是不可或缺的。

今晚上跟一桌全是撸代码的人儿吃饭，深深觉得自己见得太少，昨天把Vue.js 的框架搭好了，也可以在本地看到效果，但是，问题来了，不知道下一步该干嘛，查了很多关于 vue-cli vue-router等的意思，但是还是觉得不理解，不过之前的项目让自己对数据的双向绑定倒是理解的深一些了。现在终于明白，为什么很多企业招聘需要工作年限了，有些东西就是需要时间来慢慢积累的，急不得。所以，一步一个脚印的稳扎稳打的往前走。

把写的代码commit到GitHub之后，看了看别人写的，发现有很多想法很好，比如我当时对表单的左侧右对齐，右侧左对齐一直没想到好办法，但是看到别人代码中，把左侧的文字放到一个盒子里，右侧的文字统一放到一个盒子里，然后再排版，这种方法挺好的。以后自己写完代码之后，多找几篇实现同样功能的代码review一下，扬长避短，努力写出易读易用的好代码~

读诗时间

每到这个时候，都觉得很放松，代码和诗其实都是艺术。皆可陶冶身心。今天分享仓央嘉措的《十诫诗》, 个人非常喜欢的一首诗。

第一最好不想见，如此便可不相恋。

第二最好不相知，如此便可不相思。

第三最好不相伴，如此便可不相欠。

第四最好不相惜，如此便可不相忆。

第五最好不相爱，如此便可不相弃。

第六最好不相对，如此便可不相会。

第七最好不相误，如此便可不相负。

第八最好不相许，如此便可不相续。

第九最好不相依，如此便可不相偎。

第十最好不相遇，如此便可不相聚。

但曾相见便相知，相见何如不见时。

安得与君相诀绝，免教生死作相思。
